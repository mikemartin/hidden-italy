{{# How it works #}}
{{ partial:typography/h2 content="How sections are defined" }}

The main page builder is a single replicator field defined in resources/fieldsets/page_builder.yaml. It is imported into the page, home, tour_type and product blueprints. Sets are organised into three groups: Image and text, Interactive and Marketing sections. Each set imports its own fieldset and renders through a partial in resources/views/page_builder/.

The tour-only sections have been promoted into a dedicated shared fieldset at resources/fieldsets/tour_builder.yaml. Both guided/tour.yaml and self-guided/tour.yaml blueprints simply import: tour_builder, eliminating the duplicated YAML that used to be inlined in each. View partials are shared between contexts: highlights delegates to the main _icon_grid partial via params; testimonials and faqs import their main-builder fieldsets so there's no drift; itinerary, overview, experience, leader and essentials have their fields defined directly inside tour_builder.yaml.

Section settings convention. Every set imports either section_settings.yaml (none / light) or section_settings_with_dark.yaml (none / light / dark) at the bottom of its fieldset. The two fields — section_background + show_topography — sit at width: 50 each.

Tour-only sub-nav toggle. The sticky sub-nav at the top of every tour page lets editors hide individual sections from it via a hide_from_subnav toggle. The field is defined inline on each set inside tour_builder.yaml — not in the shared section_settings.yaml / section_settings_with_dark.yaml, which stay clean of tour-only concerns. Because it only lives in tour_builder.yaml, the toggle only appears on guided / self-guided tour entries; main-builder sets don't see it. The panel (surface, internal padding, stack-space-collapse) and the topographic decoration are applied centrally by page_builder/_block.antlers.html; per-section partials only have to pass their content-layout classes. The light panel uses bg-card, the dark panel uses the new fixed bg-card-dark token + scopes .dark. Cards rendered inside a light-panel section flip their own surface to bg-background via a card_bg scope variable so they don't disappear into the same parchment-50 panel. The dark-capable testimonial_overlap uses the symmetric .light reset class to keep its parchment quote card inside the deep-blue band. Two adjacent same-surface panels render a content-width ::before divider at the seam.

Set-picker screenshots live in public/page_builder/ and are referenced from each set's image: key with a bare filename. The status pills on each card below are auto-detected via the file_exists modifier (app/Modifiers/FileExists.php) — drop a new <handle>.png in and the pill flips to green on the next request.
