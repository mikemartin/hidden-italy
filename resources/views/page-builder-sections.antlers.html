{{# How it works #}}
{{ partial:typography/h2 content="How sections are defined" }}

The main page builder is a single replicator field defined in resources/fieldsets/page_builder.yaml. It is imported into the page, home, tour_type and product blueprints. Sets are organised into three groups: Image and text, Interactive and Marketing sections. Each set imports its own fieldset and renders through a partial in resources/views/page_builder/.

Tour-specific sections live inside the guided/tour.yaml and self-guided/tour.yaml blueprints as inline replicator sets. View partials are shared between the two contexts where the field shape allows; highlights, testimonials and faqs import their main-builder fieldsets so there's no drift, while itinerary, overview, experience, leader and essentials are still defined inline in both blueprints.

Section settings convention. Every set imports either section_settings.yaml (none / light) or section_settings_with_dark.yaml (none / light / dark) at the bottom of its fieldset. The two fields — section_background + show_topography — sit at width: 50 each. The panel (surface, internal padding, stack-space-collapse) and the topographic decoration are applied centrally by page_builder/_block.antlers.html; per-section partials only have to pass their content-layout classes. The light panel uses bg-card, the dark panel uses the new fixed bg-card-dark token + scopes .dark. Cards rendered inside a light-panel section flip their own surface to bg-background via a card_bg scope variable so they don't disappear into the same parchment-50 panel. The dark-capable testimonial_overlap uses the symmetric .light reset class to keep its parchment quote card inside the deep-blue band. Two adjacent same-surface panels render a content-width ::before divider at the seam.

Set-picker screenshots are referenced from each set's image: key. Files for main-builder sets live in public/page_builder/; tour-only set previews live in public/assets/set-previews/. Status pills on each card below show whether the referenced image exists, is missing, or points at a stale filename.
