{{# @name Page builder block @desc The page builder block wrapper #}} {{# When the block is given a `subnav_id` it doubles as a tour-subnav target. An absolutely positioned sentinel at the section's content-top owns the `x-intersect`, so active-state changes when the heading actually arrives near the sticky nav (rather than when the section's border edge merely crosses the viewport). For sections with their own `py-*` (passed via `class`), pass `sentinel_top` mirroring the padding tier — e.g. `top-12 md:top-16 lg:top-24` for `py-12 md:py-16 lg:py-24` — so the sentinel sits at the content-top after padding. Defaults to `top-0` for sections with no padding. The parameter is `subnav_id` — NOT `id` — because Statamic's page_builder iteration auto-exposes each block's UUID as `id` in scope. Naming this `id` would inherit that UUID into partials that *don't* explicitly pass one (e.g. _history, _experience), which causes the inner block to overwrite `$store.tourSubnav.active` with the UUID and break active-state matching. #}}
