{{# @name Tour summary gallery @desc Mosaic gallery for the tour overview block. Two layouts so each viewport gets a polished mosaic without shipping a stretched version of the other: mobile (2 rows, up to 5 photos) row 1 — 16/9 landscape + 1/1 square row 2 — three 1/1 squares desktop (3 rows, up to 7 photos) row 1 — 3/4 portrait + 4/3 landscape row 2 — 16/9 landscape + 1/1 square row 3 — three 1/1 squares Both layouts start at the first asset. The last visible tile becomes a "+N" overflow indicator when the gallery has more photos than the layout shows. Photos are addressed with index access (`gallery.N`) and passed to the picture tag via `:image=`. We deliberately avoid the `{{ gallery limit/offset }}` form because Statamic's shared OrderedQueryBuilder mis-handles repeated identical limit/offset calls, which a parallel mobile/desktop tree requires. Each tile is wrapped in an `` wired to lightGallery in dynamic mode, so mobile and desktop triggers share the same lightbox without registering as duplicate slides. The lightbox carries every photo in the gallery — including ones hidden behind the "+N" overflow tile — so swiping past the last visible tile reveals the rest. @param gallery The asset collection from the tour entry's `gallery` field. #}} {{ gallery_count = gallery | count }}
